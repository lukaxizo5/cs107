# RISC-V Emulator

ეს პროექტი წარმოადგენს **RISC-V ASSEMBLY** ემულატორს, რომელიც ასრულებს მულტიპლიკაციურ, არითმეტიკულ, ლოგიკურ და ბრენჩ ოპერაციებს. ემულატორი ახორციელებს ინსტრუქციების დეკოდირებას, რეგისტრების მართვასა და მეხსიერების წვდომას. კომენტარები სტანდარტულად #-ის შემდეგ.

## ემულატორი აქვს შემდეგი ინსტრუქციების მხარდაჭერა:
### არითმეტიკული:
- `add`, `sub`, `mul`, `div`, `rem`

### ლოგიკური:
- `and`, `or`, `xor`, `sll`, `srl`

### მეხსიერების მართვის:
- `lb`, `lh`, `lw`, `sb`, `sh`, `sw`

### ბრენჩის:
- `beq`, `bne`, `blt`, `bge`, `bgt`, `ble`

### immediate:
- `addi`, `xori`, `ori`, `andi`, `slli`, `srli`, `li`

### ფსევდოინსტრუქცია:
- `mv`

### jump ინსტრუქცია:
- `j`

### გამოძახება:
- `call`, `ecall`

## ტესტირების ინსტრუქცია:
ემულატორის ტესტირებისთვის შეგიძლიათ გამოიყენოთ `tests` ფოლდერში არსებული `.s` ფაილები, რომლებშიც წერია RISC-V ასემბლერის კოდები.

### ტესტირების ფაილები:

- **fact.s** – ითვლის რიცხვის ფაქტორიალს რეკურსიულად (სემინარზე დაწერილი კოდი).
- **min.s** – ითვლის მინიმალურს ორ რიცხვს შორის (ლექციაზე დაწერილი კოდი).
- **add.s** - ითვლის 5 რიცხვის ჯამს.
- **branch.s** - ითვლის 1-დან 100-მდე რიცხვების ჯამს branch-ით.
- **math.s** - ტესტავს სხვადასხვა არითმეტიკულ ინსტრუქციას.
- **mv.s** - ტესტავს mv ფსევდოინსტრუქციას.
- **store.s** და **load.s** - ტესტავენ store/load ინსტრუქციებს.

თითოეული ფაილი ტესტავს სხვადასხვა ფუნქციებს ასემბლერში. ყველა ფაილში გამოყენებულია `ecall`-ები, რომ ტესტების შედეგები ვიზუალურად იყოს შესაძლებელი.
